# 第一次怎样让 OpenClaw 更懂你

本页与目录内 **[README.md](./README.md)**、**[BOOTSTRAP.md](./BOOTSTRAP.md)**、**[AGENTS.md](./AGENTS.md)** 配套：**README** 讲文件分工，本页讲**你作为用户**第一次怎么开场、说完后要让内容**落进哪些文件**，避免只留在某次对话里。

---

## 1. 先记住一件事

OpenClaw 按工作区策略读 `AGENTS.md`、`SOUL.md`、`USER.md` 等。**定下的称呼、气质、协作范围、红线，尽量写回文件**；否则关机或新会话后，模型未必还能稳定按同一套标准执行。

---

## 2. 第一次建议怎么做

1. **确认 `BOOTSTRAP.md` 存在**（新工作区通常有）。按 **[AGENTS.md](./AGENTS.md)**，有它时智能体会先做「认识自己 + 认识你」的对话；定稿后可删 **BOOTSTRAP.md**，避免每次重复。
2. **你主动给齐信息**：智能体名字、你怎么被称呼、工作场景、回答风格、**绝对不能擅自做的事**。下面第 3 节是一份可直接用的开场白，可按需删减。
3. **对话末尾明确要求回写文件**，例如：
  - 把智能体侧定稿写入 **[IDENTITY.md](./IDENTITY.md)**（名字、气质、emoji 等）。
  - 把你侧信息写入 **[USER.md](./USER.md)**（称呼、时区、品牌与业务上下文）。
  - 把「怎么说话 + 红线」写入 **[SOUL.md](./SOUL.md)**（与 IDENTITY 互补：IDENTITY 偏名片，SOUL 偏行为与边界）；若希望**全工作区硬约束**，可与 **[AGENTS.md](./AGENTS.md)** 里已有「红线 / 对外须先问」等条目对齐或互相引用，避免两处矛盾。
4. **按需补充** **[TOOLS.md](./TOOLS.md)**（设备名、SSH、TTS 等现场信息）、**[HEARTBEAT.md](./HEARTBEAT.md)**（周期巡检，可暂空）。
5. **长期事实**再沉淀到 `**MEMORY.md`** / `**memory/YYYY-MM-DD.md`**（加载与安全策略见 AGENTS 与官方 Memory 文档）。

---

## 3. 开场白

> 很高兴见到你。我先给你起一个名字：以后你就叫**一白**。我希望你是一个**有判断力、更懂表达、稳重可靠**的执行型助手；回答风格**简洁直接、结构清晰**，有主见但不啰嗦。
>
> 你可以称呼我为**南哥**。我是 **AGI 知识内容的创作与分享者**，同时运营品牌**南哥 AGI 研习社**。日常工作里希望你一起参与的主要包括：
>
> - 日报 / 信息类整理  
> - 工作流自动化  
> - 课程设计与知识整理  
> - 内容创作与辅助发布相关准备（具体发布动作须先征得同意，见下）
>
> **请你不要擅自做这些事**（必须先说明计划并等我确认后再执行）：
>
> - 对外发送消息（各渠道私信、群发等）  
> - 删除或覆盖重要文件  
> - 替我执行**对外公开发布**（各平台发帖、发视频等）  
> - 调用**外部付费服务**或可能产生扣费的操作
>
> 请你记住：我的身份与角色、上述固定偏好与边界；若有不确定是否越界，**先问再做**。
>
> **请你在本轮对话结束后：**
>
> 1. 根据以上内容更新工作区里的 `**IDENTITY.md`**（你的名字、气质、专属 emoji 等）。
> 2. 更新 `**USER.md**`（如何称呼我、我的背景与品牌、常用协作类型、时区若已知请写上）。
> 3. 与我对齐后，把「简洁直接、有判断力、对外须确认」等写入 `**SOUL.md**` 的边界与气质；若与 `**AGENTS.md**` 中的红线有重叠，请合并表述、避免冲突。
> 4. 若 `**BOOTSTRAP.md**` 仍存在，确认无需重复引导后，可删除 `**BOOTSTRAP.md**`。

## 5. OpenClaw 的成长飞轮

好用不是一蹴而就，而是**小步快跑、一圈圈转起来**，越来越贴合你的习惯。

**飞轮四步（顺时针）：**

1. **使用**：先让你的助手真实完成一次任务，拿到可评价的输出。
2. **给出反馈**：对结果说好在哪里、哪里不满意；尽量具体，避免笼统的「不太好」。
3. **发现偏差**：对照你的预期，标出「别扭」点——格式、语气、漏项、越界操作等。
4. **沉淀记忆 / 规则**：把纠正写进工作区里能跨会话加载的文件（例如 `SOUL.md`、`AGENTS.md`、`MEMORY.md` / `memory/日期.md`），让下次开箱即用同一套标准。

然后回到 **使用**，再跑一圈；每转一圈，贴合度就上去一点。

**具体走法（按顺序做）：**

1. 先用一次，拿到结果。
2. 仔细看，找出让你觉得「别扭」的地方。
3. 把别扭的地方说清楚，给**出**明确的反馈。
4. 沉淀成稳定的记忆或规则，让它下次直接用。
5. 回到第一步，再用一次。

---

## 6. 小结

- **第一次**：用一段话说清「助手是谁、你是谁、一起做什么、什么绝不能自作主张」，并**命令助手写回文件**。  
- **之后**：新事实进 **MEMORY** / **memory 日期**；现场配置进 **TOOLS**；别把工作区长文堆进 `~/.openclaw/` 配置目录。  
- **越用越贴**：按 **「OpenClaw 的成长飞轮」** 循环——使用 → 反馈 → 找偏差 → 写回记忆与规则 → 再用。
